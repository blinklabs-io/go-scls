# go-scls Agent Guide

Pure-Go library for the Cardano Standard Canonical Ledger State (SCLS)
container format defined in [CIP-0165](https://github.com/cardano-foundation/CIPs/tree/master/CIP-0165):
read, write, and verify streaming ledger-state snapshot files. Primary agent
reference; Claude-specific layer in `CLAUDE.md`.

## Commands

| Action | Command |
|--------|---------|
| Tests + race | `make test` |
| Format | `make format` |
| Nilaway | `make nilaway` |
| Fuzz the verifier | `go test -run=NONE -fuzz=FuzzVerify -fuzztime=2m .` |
| Conformance | `go test -run TestConformance -v .` (add `SCLS_CONFORMANCE_FILE=/path/to/file.scls` for external fixtures) |

All must pass before submitting. Apache-2.0 header (Blink Labs Software) on
every Go file. Conventional commit messages; GPG-signed commits.

## Layout

Single flat package `scls` at the repo root (Blink Labs convention), layered
internally:

```text
hash.go      Blake2b-224 digests, 0x00/0x01 domain separation
merkle.go    incremental unbalanced Merkle tree (promote-then-merge)
wire.go      record framing: size u32 BE || type u8 || payload
header.go    HDR record (magic "SCLS", version u32)
entry.go     length-prefixed entries (raw fixed-size key + verbatim value)
chunk.go     CHUNK record codec + footer hash
manifest.go  MANIFEST record codec (binary fields, tstr summary, offset bookend)
manifest_locate.go  ReadManifest: seek to the manifest via the offset bookend
writer.go    streaming Writer (ordering enforcement, count/size chunk rotation)
reader.go    streaming Reader (skips unknown record types)
verify.go    Verify with 3 levels; FuzzVerify + FuzzRoundTrip targets
spec/        vendored CIP-0165 spec + RECONCILIATION.md (wire-format decisions)
testdata/    cross-implementation conformance fixtures
```

## Invariants that are easy to get wrong

1. The u32 record size prefix covers the **type byte + payload** (HDR record
   size is 9, not 8). Zero-size records are invalid.
2. **No CBOR anywhere in the container framing.** Namespaces and summary
   strings are u32-length-prefixed UTF-8; entry keys are raw fixed-size
   bytes. Entry *values* are caller-supplied CBOR written verbatim — never
   re-encode them. This library deliberately has no CBOR dependency.
3. Entry digest preimage is `Blake2b-224(0x01 || ns || key || value)` over
   raw bytes; chunk hash is `Blake2b-224` of the concatenated entry digests
   (no domain prefix); global-tree leaves are `Blake2b-224(0x01 || ns_root)`
   with the namespace name excluded.
4. Chunk sequence numbers are **per-namespace** and reset on namespace
   change. The start value is not pinned by the spec (Rust writes from 0,
   Haskell from 1) — verification only requires strict increase within a
   namespace.
5. Manifest `namespace_info` order is **not significant**; compare as a set
   keyed by name. The Merkle global root always uses lexicographic namespace
   order. The manifest payload ends with a u32 offset equal to the record's
   own size prefix (the bookend used to find the manifest from file end).
6. Every wire-format decision has a numbered row in `spec/RECONCILIATION.md`
   with sources; code comments reference rows as `VERIFY(#n)` /
   `RECONCILIATION #n`. When the spec or a reference implementation
   contradicts this library, update the table with sources first.
7. Pure Go, no CGO. Only direct dependency: `golang.org/x/crypto`. Do not
   add gouroboros (it may depend on this library later; avoid the cycle).

## Out of scope for v0.1 (reject, don't implement quietly)

zstd chunk formats (`0x01`/`0x02` → `ErrUnsupportedChunkFormat`), DELTA
records (`ErrUnexpectedRecord`), multi-manifest/appended files, namespace
payload schemas, proof generation.

## Reference implementations

- Haskell (canonical): https://github.com/tweag/cardano-cls
- Rust: https://github.com/tweag/cardano-scrawls — `src/records/*.rs` and
  `src/hash/merkle.rs` are the fastest way to answer wire-format questions.
- `testdata/minimal-raw.scls` was generated by the Haskell tool; it must
  always pass `Verify(..., VerifyFull)`.
